* Introduction to Python Interaction
- Python can run interactively through command line (Windows, Mac, Linux)
- Python shows a prompt (>>>), waiting for instructions
- Python does nothing until given commands by the user
- Python follows the "what next?" model, waiting for instructions
- Users type statements directly to control Python execution

* Assignment Statements and Variables
- Example: =x = 1= stores value 1 in variable x
- Assignment is directional (store value into memory location)
- Variables represent named memory locations
- =print(x)= retrieves value stored in variable x
- Expressions like =x = x + 1= update variable values
- Variables allow storing and reusing data in programs

* Expressions and Syntax Basics
- Expressions perform operations like addition (=+=)
- Operators include =+= and ===
- Functions like =print()= execute actions
- Parameters pass values into functions
- Syntax errors occur when Python cannot understand commands

* Programming as Language Structure
- Programming is like writing: alphabet → words → sentences → paragraphs
- Programs are built from small components into larger structures
- Lines of code act like sentences
- Groups of lines form logical blocks (paragraphs)
- Entire programs form complete "stories" solving problems

* Reserved Words Concept
- Reserved words have fixed meanings in Python
- Examples: =if=, =for=, =while=, =import=, =assert=
- Reserved words cannot be used as variable names
- Python reacts strongly to reserved words when encountered
- Analogy: reserved words are like commands a dog recognizes ("walk", "food")

* Writing Code in Scripts vs Interactive Mode
- Interactive mode is useful for quick testing
- Writing long programs interactively is inefficient
- Scripts are stored Python programs in =.py= files
- Scripts allow running multiple lines automatically
- Text editors (e.g., Atom) help write and debug scripts
- Syntax highlighting improves readability and error detection

* Basic Programming Patterns Overview
- Programs are built using a few core patterns
- Main patterns: sequential, conditional, repetition, store/retrieve
- These patterns are combined to create complex programs

* Sequential Execution Pattern
- Sequential: execute steps one after another
- Example: assign → print → update → print
- Program flows line by line in order
- Flowcharts can represent execution order visually

* Conditional Execution Pattern
- Conditional uses =if= statement to make decisions
- Condition evaluates to True or False
- If True → execute indented block
- If False → skip block
- Example: =if x < 10:= then print "smaller"
- Indentation defines which code belongs to condition
- Multiple conditionals can exist in one program

* Repetition (Looping) Pattern
- Loops repeat actions multiple times
- Common loop keywords: =while=, =for=
- =while= loop repeats while condition is True
- Loop checks condition before each iteration
- Example: count down from 5 to 1 using loop
- Loop exits when condition becomes False
- Loop variable changes each iteration to avoid infinite loops
- Infinite loops occur if condition never becomes False

* Iteration and Loop Control
- Iteration variable controls loop execution
- Variable is initialized, checked, and updated each loop
- Example: =n = n - 1= reduces value each iteration
- Loop stops when condition fails (e.g., n = 0)

* Nested and Combined Structures
- Programs combine sequential, conditional, and loop patterns
- Nested structures place one block inside another
- Example: =for= loop inside another =for= loop
- Example: =if= statement inside a loop
- Indentation shows nesting structure

* Example Program Structure (Word Count)
- Program reads a file name from user
- Opens file and processes text
- Builds histogram of word frequencies
- Updates count for each word encountered
- Uses dictionaries to store word counts
- Iterates through histogram to find most frequent word
- Outputs most common word and count

* Program Organization as Paragraphs
- Programs can be divided into logical sections (paragraphs)
- First section: input and file handling
- Second section: processing data (word counting)
- Third section: analyzing results (finding maximum)
- Each section has a clear purpose in the program flow
